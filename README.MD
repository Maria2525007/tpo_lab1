### Allure-отчёт

- **Локально:** после тестов сгенерировать отчёт с историческим трендом (последние 10 сборок):
  ```bash
  chmod +x scripts/allure-report-with-history.sh
  ./scripts/allure-report-with-history.sh
  allure open target/allure-report
  ```
  Для тренда нужен [Allure CLI](https://allurereport.org/docs/getting-started/). Без CLI: `mvn test allure:report` и открыть `target/allure-report/index.html`.

- **Только генерация отчёта после прогона:** `mvn test io.qameta.allure:allure-maven:2.17.0:report` — отчёт в `target/allure-report`. Короткая форма `mvn allure:report` работает, если в `~/.m2/settings.xml` в `<pluginGroups>` добавлен `io.qameta.allure`.

- **Разметка тестов:** `@Feature` (Математика / Алгоритмы / Доменная модель), `@Story`, `@Severity` (Critical/Blocker), `@Link` (ссылка на требование). При падении теста в отчёт добавляется вложение с текстом ошибки.

- **CI:** в пайплайне (GitHub Actions) тесты запускаются, генерируется Allure-отчёт, история сохраняется в кэше для тренда. Артефакт `allure-report` можно скачать со вкладки Actions.
